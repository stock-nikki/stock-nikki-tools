# クロちゃんへの指示：ポートフォリオ損益レポート グラフHTML更新

## 概要
添付の3ファイルを使って、stock-chart-all.html のデータブロック（BLOCK 1〜6）を更新してください。

## 添付ファイル
1. **stock-chart-all.html** — 現在のグラフHTML（これを更新する）
2. **SBI証券スクリーンショット** — 最新のポートフォリオ画像
3. **セクター騰落率TSV** — チャッピーが取得した東証33業種の騰落率データ

## 更新対象と手順

### BLOCK 1: HISTORY（週次推移）
- SBIスクショ最下部の合計行から「評価額合計」「含み損益合計」を読み取る
- 投資額 = 評価額合計 − 含み損益合計
- 末尾に1行追記（weekラベルは当日の日付を `M/D` 形式で）
- 小数がある場合はそのまま保持（例: value: 306057.2）

### BLOCK 2: STOCKS（銘柄別損益・前日比）
- SBIスクショから全銘柄の以下を読み取る：
  - `pnl` = 損益 列の値
  - `pct` = 損益(%) 列の値
  - `chg` = 前日比(%) 列の値（前日比の円額ではない）
- 損益率（pct）の降順で並べる（プラス→マイナス）
- 新規購入銘柄があれば追加、売却済みがあれば削除

### BLOCK 3: STOCKS_HEATMAP（銘柄別評価額）
- SBIスクショから全銘柄の以下を読み取る：
  - `value` = 評価額 列の値（右端の列。時価総額ではない）
  - `pnl` = 損益 列の値（BLOCK 2と同値）
  - `pct` = 損益(%) 列の値（BLOCK 2と同値）
- 評価額（value）の降順で並べる

### BLOCK 4: WEEKS（週ラベル）
- 末尾に当日の日付ラベルを追加（BLOCK 1のweekと揃える）

### BLOCK 5: SECTORS（セクター別pnl配列）
- 各銘柄のpnl配列の末尾に、今回のpnl値を追加
- 新規銘柄は過去の週を `null` で埋めてからpnlを追加
- 新規銘柄のセクター分類は既存の分類に合わせる。不明なら聞いてください
- 配列の長さは全銘柄でWEEKSの長さと一致すること

### BLOCK 6: MKT_RAW（市場セクタートレンド）
- 添付のセクター騰落率TSVの値で全項目を上書き
- プロパティ対応: `d`=前日比, `w`=1週間比, `m`=1ヶ月比, `h`=6ヶ月比, `y`=1年比
- TSVに「N/A」がある場合は `0` をセットし `// ※TSVでN/A` とコメントを付ける

## ルール
- `// 🔄` から `// 🔄 ここまでが毎週更新エリア` の範囲だけを編集する
- `// ⛔ 以下、描画ロジック` 以降は変更しない
- `// 📌 静的設定` は新セクター追加時のみ変更（通常は触らない）
- 読み取れなかった値がある場合は `null` ではなく「要確認」コメントを付けて聞いてください
- 更新済みHTMLをファイルとして出力してください

## 検証チェックリスト（更新後に必ず実行）
出力前に以下をスクリプトで検証し、結果を報告すること：

1. **構文チェック**: データブロック全体がJavaScriptとしてパースできること
2. **銘柄数一致**: STOCKS / STOCKS_HEATMAP / SECTORSの全銘柄が同数で、名前が完全一致すること
3. **配列長一致**: SECTORSの全銘柄のpnl配列長 = WEEKSの長さ = HISTORYのエントリ数
4. **合計値一致**: STOCKS_HEATMAPのvalue合計 ≒ HISTORYの最新value（差額1円以内）
5. **ソート順**: STOCKSがpct降順、STOCKS_HEATMAPがvalue降順であること
6. **更新範囲外が無変更**: 🔄ブロック外のコードが元ファイルと完全一致すること

※1つでもNGがあれば修正してから出力する
