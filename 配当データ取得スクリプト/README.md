# nikkei225_dividend_fetcher

日本株の配当・優待データを一括取得するPythonスクリプト。  
yfinance（配当・財務）と、とあるサイトｗ（株主優待）から1銘柄ずつ取得し、TSVに出力します。

ブログ「[Sから始まるStock日記](https://stock-nikki.com)」のスクリーニング用に開発。

## 特徴

- **2ソース並行取得** — yfinance（配当・PER・PBR等）→ とあるサイト（優待情報）を1銘柄ずつ交互に実行
- **異常値検証** — 配当利回りの自前計算・raw値との乖離チェック・注意フラグ自動付与
- **途中再開対応** — `--resume` で前回の出力TSVから未取得・エラー銘柄だけ再取得
- **行範囲指定** — `--s` / `--e` で数千銘柄を複数セッションに分けて実行可能
- **ブロック回避** — UAローテーション・3〜5秒ランダム待機・連続エラー検知で自動停止

## セットアップ

```bash
pip install yfinance pandas requests beautifulsoup4
```

## 入力ファイル

### 銘柄リストTSV（デフォルト: `stock_list.tsv`）

タブ区切り。ヘッダーに「コード」を含む列と「銘柄」を含む列があれば自動認識します。  
JPXの銘柄一覧ExcelをTSV化すればOK。

```
コード	銘柄名	33業種区分
7203	トヨタ自動車	輸送用機器
6758	ソニーG	電気機器
9984	ソフトバンクG	情報・通信業
```

以下のJPX拡張カラムがあれば自動で読み込み、出力TSVにも引き継ぎます:

- 市場・商品区分 / 33業種コード / 33業種区分 / 17業種コード / 17業種区分 / 規模コード / 規模区分

## 使い方

```bash
# 基本（stock_list.tsv の全銘柄を取得）
python nikkei225_dividend_fetcher.py

# 銘柄リストを指定
python nikkei225_dividend_fetcher.py --list standard225.tsv

# 行範囲を指定して実行（ヘッダー込みの行番号）
python nikkei225_dividend_fetcher.py --s 1 --e 500
python nikkei225_dividend_fetcher.py --s 501 --e 1000

# 片方だけの指定もOK（--s だけなら末尾まで、--e だけなら先頭から）
python nikkei225_dividend_fetcher.py --s 501
python nikkei225_dividend_fetcher.py --e 500

# 前回の出力TSVから途中再開
python nikkei225_dividend_fetcher.py --resume dividend_20260625.tsv
```

### コマンドオプション

| オプション | 説明 | デフォルト |
|---|---|---|
| `--list FILE` | 銘柄リストTSVファイルを指定 | `stock_list.tsv` |
| `--s N` | 開始行番号（ヘッダー込み1-indexed、省略時は先頭から） | 全件 |
| `--e N` | 終了行番号（ヘッダー込み1-indexed、省略時は末尾まで） | 全件 |
| `--resume FILE` | 前回の出力TSVを指定して途中再開 | なし |

### 大規模取得のコツ

JPX全銘柄（約3,700件）など大量に取得する場合は、`--s` / `--e` で分割実行 → `--resume` で未取得分を回収するのがおすすめです。

```bash
# Day 1: 1〜1000件目
python nikkei225_dividend_fetcher.py --s 1 --e 1000

# Day 2: 1001〜2000件目
python nikkei225_dividend_fetcher.py --s 1001 --e 2000

# Day 3: 残り
python nikkei225_dividend_fetcher.py --s 2001 --e 3735

# エラー分を再取得
python nikkei225_dividend_fetcher.py --resume dividend_20260625.tsv
```

## 出力

### TSVファイル（`dividend_YYYYMMDD.tsv`）

銘柄リストの入力順で出力されます。

| カラム | ソース | 説明 |
|---|---|---|
| コード | 入力TSV | 証券コード |
| 銘柄名 | 入力TSV | 銘柄名 |
| 市場・商品区分 〜 規模区分 | 入力TSV | JPX拡張カラム（あれば） |
| 株価 | yfinance | 現在株価 |
| 配当利回り(%) | 自前計算 | 1株配当÷株価×100 |
| 1株配当(円) | yfinance | 年間1株配当 |
| 配当利回り_raw | yfinance | dividendYield（参考値） |
| 1株配当_raw | yfinance | dividendRate（参考値） |
| 配当利回り_自前計算 | 自前計算 | 検証用 |
| PER / PBR | yfinance | 株価指標 |
| 時価総額(億円) | yfinance | 時価総額 |
| 52週高値 / 52週安値 | yfinance | 52週レンジ |
| セクター | yfinance | セクター（英語） |
| 優待有無 | とあるサイト | あり / なし / 要確認 |
| 優待内容 | とあるサイト | 優待の内容（最大300文字） |
| 優待最低株数 | とあるサイト | 優待取得に必要な最低株数 |
| 優待判定信頼度 | とあるサイト | 高 / 低 |
| 注意フラグ | 検証 | 異常値疑い・高利回り要確認 等 |
| 要確認理由 | 検証 | フラグの詳細理由 |
| ランキング利用可否 | 検証 | OK / 要確認 / 除外推奨 |
| yfinanceの状態 | - | OK / OK_高利回り要確認 / エラー 等 |
| とある優待サイトの状態 | - | OK / 403 ブロック / タイムアウト 等 |

### エラーログ（`dividend_error_YYYYMMDD.log`）

取得エラーの詳細を記録。とある優待サイトの403ブロックやyfinanceのデータなし等をトラブルシュート用に保持。

## 異常値検証

配当データの信頼性を自動チェックし、3段階でフラグを付与します。

| 注意フラグ | 条件 | ランキング利用可否 |
|---|---|---|
| 高利回り要確認 | 利回り 8%以上 | 要確認 |
| 異常値疑い | 利回り 15%以上 | 要確認 |
| 配当かなり異常値疑い | 利回り 30%以上 or 1株配当が株価の30%超 | 除外推奨 |
| 利回り乖離 | raw値と自前計算の差が2%以上 | - |

## 中断・再開

`Ctrl+C` で中断しても、取得済みデータはTSVに保存されます。  
`--resume` で同じTSVを指定すれば、yfinance状態・とある優待サイト状態が「OK」以外の銘柄だけ再取得します。

## 注意事項

- とあるｗ優待情報サイトへのアクセスはウェイト（3〜5秒）を入れています（重要！！外さないでください！！）が、短時間に大量取得するとブロックされる可能性があります。
- 5連続エラーで警告、10連続エラーで優待取得を自動停止します。
- yfinanceの配当データは速報値のため、正確な値は証券会社や四季報で要確認です。
- 【重要】そもそもこのスクリプトが、、、と言うのはありますが、短時間に大量アクセスしてご迷惑をかけないようにしましょう。
